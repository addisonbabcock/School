Inserting Data into a table with an Identity Column

DO NOT supply a value for an identity column!  SQL Server will create a value automatically!

To demonstrate, consider the following table:

Create Table Supplier (

/* create SupplierId col, make it identity col and pri key: */
SupplierId int identity (1, 1) not null
   Constraint PK_Supplier Primary Key clustered,

Name varchar(100)  not null,
Phone char(14) not null
Constraint CK_Phone
)

To INSERT DATA, DO NOT SUPPLY A VALUE FOR THE IDENTITY COL, IE:

INSERT Supplier (Name,Phone)
VALUES  ('Smith Supplies','471-7400')
