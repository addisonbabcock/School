Syntax for creating a composite Foreign Key

..by example:

CONSTRAINT FK_Registration_ClassOrGroup_ClassOrGroup FOREIGN KEY
   (ClassName,Year,SemesterCode) REFERENCES Classes (ClassName,Year,SemesterCode)

The script above can be used at table creation time (or after a table is created using ALTER TABLE) to specify a composite FK constraint called "FK_Registration_ClassOrGroup_ClassOrGroup".

It is an ugly name, but very descriptive reflecting the columns involved!

In this example, the columns "ClassName", "Year", and "SemesterCode" are specified to be a FK to columns with the same names in the parent table called "Classes".
